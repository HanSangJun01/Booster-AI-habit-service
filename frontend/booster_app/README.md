# Booster (Flutter)

부스터 HTML 목업을 **하나의 Flutter 앱**으로 합친 버전이에요.
하단 5탭(홈 / 팀 / 인증 / 상점 / 마이페이지)으로 모든 화면이 연결되고,
탭별로 독립된 네비게이션 스택을 가져서 유저플로우대로 화면이 쌓입니다.

## 실행 방법

이 폴더에는 `lib/`, `pubspec.yaml`, `assets/`만 들어 있어요.
플랫폼 폴더(android/ios 등)는 아래 명령으로 한 번 생성하면 됩니다.

```bash
cd booster_app
flutter create .          # 현재 폴더에 android/ios/web 등 플랫폼 스캐폴딩 생성
flutter pub get
flutter run               # 기기/시뮬레이터 선택 후 실행
```

> 외부 패키지 없이 **Flutter SDK 기본 기능만** 사용했어요(아이콘은 Material 내장).
> 이 환경에서 컴파일 검증은 못 했으니, 첫 빌드에서 사소한 수정이 필요할 수 있어요.

## 화면 / 유저플로우

```
홈 탭
  홈(진행 중 / 복귀 / 생성 전 — 상단 [데모] 칩으로 전환)
    └ 생성 전 → "챌린지 만들기" → 개인 챌린지 생성 → 완료 시 진행 중
    └ "오늘 인증하러 가기" → 인증 탭으로 전환

팀 탭
  팀 홈(내 팀 목록 / 빈 상태 — [데모] 칩으로 전환)
    ├ 팀 카드 탭 → 팀 상세 → "참여하기" → 팀 배틀 → "인증하러 가기"(인증 탭)
    ├ 팀 탐색하기 → 탐색(공개 팀만) ─ 카드 탭 → 팀 상세
    │                            └ "코드 참여" → 방 코드 입력 시트
    └ 팀 만들기 → (1) 기본정보 → (2) 공개/비공개
                     └ 공개  → 생성 후 팀 홈 복귀
                     └ 비공개 → 방장 화면(방 코드 7K2Q · 복사/공유 · 모집현황)

인증 탭 (가운데 FAB)
  오늘의 인증 — 개인 챌린지 "인증하기" → GPS 오버레이(탐지→매칭→성공)
              팀 챌린지 카드는 "곧 제공" 토스트

상점 탭   준비 중 플레이스홀더
마이페이지 프로필 / 내 활동 / 설정(토글) / 계정
```

## 폴더 구조

```
lib/
  main.dart                       앱 진입점
  theme/booster_theme.dart        색·그라데이션·텍스트 토큰(BC)
  widgets/common.dart             공통 위젯(코인칩·헤더·카드·칩·버튼·토스트)
  screens/
    main_scaffold.dart            5탭 IndexedStack + 탭별 Navigator + 하단 내비/FAB
    home/        home_screen, personal_create_screen
    team/        team_home, team_explore, team_detail, team_battle, team_create(+방장)
    verify/      verify_screen (GPS 오버레이)
    shop/        shop_screen
    mypage/      mypage_screen
assets/booster_emblem.png         로고 엠블럼
```

## 알아둘 점 (목업 → 앱 변환 시 결정/차이)

- **상태 관리**는 외부 패키지 없이 `StatefulWidget`만 사용했어요(프로토타입 단순화·pub 의존 최소화).
- **아이콘**은 목업의 인라인 SVG/Tabler 대신 **Material 내장 아이콘**으로 대체했어요(자체 포함·무의존).
- **팀 모델 표기 불일치**: 새로 정한 팀 흐름(공개/비공개+방코드, 인원 3:3·4:4·5:5 가변, 반경 제거)은
  생성 플로우에 반영했지만, 기존 **팀 상세/배틀 화면 문구**는 예전 "10명 랜덤 5:5" 톤이 일부 남아 있어요.
  상세 화면 대결방식 문구는 "인원이 모이면 두 팀으로 나뉘어"로 살짝 다듬어 뒀고,
  정확한 카피는 확정해 주시면 통일하겠습니다.
- 개인 챌린지 생성에는 목업대로 **GPS 반경(50/100/200m)** 을 남겨 뒀어요(반경 제거는 팀 챌린지 한정 결정이라).
  개인도 제거하길 원하면 알려주세요.
- 데모 편의를 위해 홈·팀 상단에 **[데모] 상태 전환 칩**을 넣어 뒀어요(빈 상태/복귀 등 확인용). 실제 빌드에선 제거하면 됩니다.

## 제품 개요

Booster는 **팀 기반 습관 챌린지 서비스**다.  
사용자는 챌린지에 예치금 코인을 걸고 상대 팀과 일일 인증 참여율을 경쟁한다. 이긴 팀이 상대 팀의 예치금을 가져가는 구조로, 개인 연속 출석 스트릭과 코인 인센티브가 습관 지속의 동기를 만든다.

**기술 스택**
- 모바일: Flutter (iOS/Android)
- 백엔드: Spring Boot REST API
- DB: PostgreSQL
- 스토리지: File/Object Storage (Phase 2 사진 인증용)

## 핵심 비즈니스 로직

### 1. 챌린지·팀 구조

- 챌린지 = 카테고리 + 제목 + 인증 방법(GPS) + 기간(day) + 예치금 + 공개/비공개 + 자동/방장 승인
- 10명이 참여 신청을 마치면 서버가 랜덤으로 5:5 팀을 구성하고 챌린지 자동 시작
- 챌린지 시작 후 나가기 없음 (회원 탈퇴·비활성화 예외만 처리)

### 2. 팀 참여율 계산

```
참여율 = 팀 전체 인증 횟수 합산 / (챌린지 기간 × 팀 인원)
```

- 탈퇴자 발생 시: 탈퇴 전 기간은 기존 인원, 이후 기간은 잔여 인원으로 분모 분리 재계산
- 탈퇴자 예치금은 반환 없음

### 3. 챌린지 종료 정산

| 결과 | 처리 |
|------|------|
| 승팀 | 전체 모금 코인 / 승팀 인원수 = 1인당 지급 |
| 패팀 | 예치금 소멸 |
| 동률(DRAW) | 전원 예치금 반환, 추가 보상 없음 |

### 4. 인증 흐름 분리 (중요)

개인 습관 인증과 팀 챌린지 인증은 **별도 흐름**으로 관리한다.

**개인 습관 인증** (스트릭·코인·복귀미션 기준)
- 성공: 스트릭 +1, 홈 대시보드·캘린더 갱신
- 미완료: 즉시 실패 아님 → 복귀 미션 대기 상태 전환
  - 복귀 성공(다음 날 23:59 이내): 스트릭 유지, **-50코인**
  - 복귀 실패: 스트릭 초기화, **-100코인**

**팀 챌린지 인증** (팀 참여율·승패 기준)
- 성공: 해당 날짜 팀 참여 기록, 팀 상세 갱신
- 미완료: 팀 참여율 분자 미포함
- 팀 챌린지 인증 미완료가 개인 스트릭을 직접 초기화하지 않음

### 5. 코인 규칙

| 사건 | 코인 변화 |
|------|----------|
| 신규 가입 | +500 (초기 지급) |
| 7일 연속 인증 달성 | +100 |
| 복귀 미션 성공 | -50 (패널티 경감) |
| 복귀 미션 실패 (최종) | -100 + 스트릭 초기화 |
| 챌린지 참여 | -예치금 (차감) |
| 챌린지 승리 | +모금 코인 / 승팀 인원 |
| 챌린지 패배 | 예치금 소멸 |
| 챌린지 DRAW | 예치금 반환 |

### 6. GPS 인증 위치 등록

- 등록 시점: 챌린지 참여 신청 시 (참여자별 개별 등록)
- 등록 정보: 위도, 경도, 반경(meters), 장소명
- 인증 성공 조건: 현재 위치가 등록 반경 내에 있을 것
- 인증 가능 시간: KST 00:00~23:59 (시간 창 제한 없음)
- 챌린지 시작 후 위치 변경 불가

## 도메인 모델 (핵심 엔티티)

| 엔티티 | 설명 |
|--------|------|
| `User` | 사용자 계정, 코인 잔액, 누적 출석 |
| `Challenge` | 챌린지 설정 (기간, 예치금, 인증 방법 등) |
| `Team` | 챌린지 내 경쟁 단위 (A팀/B팀) |
| `ChallengeParticipant` | 참여자별 챌린지 소속 + GPS 등록 위치 |
| `PersonalCheckIn` | 개인 습관 인증 기록 (스트릭·코인·복귀미션 기준) |
| `ChallengeCheckIn` | 팀 챌린지 인증 기록 (팀 참여율·승패 기준) |
| `RecoveryMission` | PersonalCheckIn 실패 후 복귀 미션 (1:1) |
| `CoinTransaction` | 코인 차감/지급 내역 |
| `Streak` | 사용자별 현재 스트릭 + 최고 기록 |

## 구현 우선순위 (권장)

### Phase 1 — MVP

1. **Backend 먼저**
   - Auth/User → Challenge/Team → GPS Check-in (PersonalCheckIn + ChallengeCheckIn) → Coin 정산
2. **Frontend 병렬 (백엔드 API 확정 후)**
   - Flutter 4탭: 홈, 팀, 인증, 마이페이지
3. **AI Service**: Phase 2 이후 (GPS로 MVP 충분)

### Phase 2 — 이후 기능

- AI 사진 인증 (AI Verification Module)
- 푸시 알림 (FCM 등)
- 소셜 로그인 (Google, Apple)
- 이메일 인증 · 비밀번호 재설정
- 인증 위치 변경 (방장 승인 방식)
- 코인 충전·결제

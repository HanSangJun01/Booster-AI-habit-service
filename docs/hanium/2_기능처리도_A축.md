## | 기능 처리도(기능 흐름도) — 샘플 1

### 기능 흐름도  〈개인 GPS 일일 인증 처리 흐름도〉

```
[시작] 사용자: POST /api/personal/check-in  { lat, lng } + JWT
   │
   ▼
① 기준 위치 등록 여부 확인 ──(없음)──▶ 400 LOCATION_NOT_REGISTERED ─▶[종료]
   │ (있음)
   ▼
② 당일 인증 기록 조회 ──(이미 SUCCESS)──▶ 409 DUPLICATE_CHECK_IN ─▶[종료]
   │ (없음 또는 RECOVERY_PENDING)
   ▼
③ GPS 반경 판정 ◇ (Haversine 거리 ≤ 등록 반경?)
   │
   ├──(반경 밖)──▶ 400 GPS_OUT_OF_RANGE  ※실패 시 레코드 생성 안 함 ─▶[종료]
   │ (반경 안)
   ▼
④ 인증 기록 확정 (기존 레코드 markSuccess / 없으면 신규 success 저장)
   ▼
⑤ 스트릭 +1 (currentStreak↑, maxStreak 갱신)   ── streaks DB
   ▼
⑥ 출석일수 +1 (totalAttendance)               ── users DB
   ▼
⑦ 보상 마일스톤 판정 ◇ (currentStreak % 보상주기 == 0?)
   ├──(해당)──▶ 코인 지급(+) STREAK_REWARD       ── coin_transactions DB
   │ (미해당)
   ▼
⑧ 응답 반환 { SUCCESS, 인증시각, 스트릭, 코인잔액, rewardGranted } ─▶[종료]
```

| ① | 기준 위치 미등록 시 인증 불가 (먼저 위치 등록 필요) |
|---|---|
| ② | `UNIQUE(user_id, check_in_date)` + 애플리케이션 검사로 하루 1회 인증 보장 |
| ③ | Haversine 거리로 등록 반경 이내 여부 판정 (외부 지도 API 호출 없음) |
| ④~⑦ | 단일 `@Transactional` — 인증·스트릭·출석·코인이 함께 반영(부분 반영 방지) |
| ⑧ | 갱신된 스트릭·코인 잔액을 응답으로 반환 |

| **프로그램 ID** | CHK-01_PersonalCheckIn |
|---|---|
| **프로그램 명** | 개인 GPS 일일 인증 |
| **작성일** | 2026. .  . |
| **Page** | 1/2 |
| **개요** | 사용자의 현재 GPS 좌표가 등록 위치 반경 이내인지 Haversine 거리로 판정하여, 성공 시 스트릭·출석·코인 보상을 단일 트랜잭션으로 처리하는 개인 습관 인증 프로그램 |

## | 기능 처리도(기능 흐름도) — 샘플 2

### 기능 흐름도  〈복귀 미션 생애주기 흐름도〉

```
[매일 배치 — DailyMissionScheduler]
   │
   ▼
①단계 expireOverdueMissions(): 마감(deadlineAt) 초과 PENDING 미션 일괄 처리
   └─ 미션 FAILED + 미인증일 FAILED + 코인 -100(클램핑) + 스트릭 reset(0)
   │
   ▼
②단계 generatePendingForYesterday(): 어제 미인증 '활성' 사용자 대상
   └─ PersonalCheckIn(RECOVERY_PENDING) 생성 + RecoveryMission(PENDING, 마감=오늘 23:59:59)
       ◇ 멱등 — 어제 레코드가 이미 있으면 건너뜀
   │
   ▼
[사용자] GET /api/personal/recovery/status → 대기 미션 확인
   │
   ▼
[사용자] POST /api/personal/recovery  { lat, lng }
   │
   ├─◇ 마감 초과? ──▶ 400 RECOVERY_EXPIRED
   ├─◇ GPS 반경 밖? ──▶ 400 GPS_OUT_OF_RANGE
   │ (마감 이내 & 반경 안)
   ▼
   미션 COMPLETED + 미인증일 SUCCESS 보정 + 코인 -50(클램핑)
   + 출석 +1 + 스트릭 keepAlive(증가 없이 유지) ─▶[종료]
```

| 1 | (배치①) 마감을 넘긴 복귀 미션을 실패 처리하고 스트릭을 초기화한다 |
|---|---|
| 2 | (배치②) 어제 미인증 사용자에게 복귀 미션을 자동 생성한다(멱등) |
| 3 | (사용자) 마감 이내 GPS 재인증 시 미인증일을 SUCCESS로 보정하고 스트릭을 유지한다 |

#### 보상/패널티 정책 요약
| 상황 | 코인 | 스트릭 | 비고 |
|------|------|--------|------|
| 일일 인증 성공 | 마일스톤 시 +보상 | +1 | 정상 출석 |
| 복귀 미션 성공 | -50 (잔액까지만) | 유지(증가 없음) | 놓친 날 보정 |
| 복귀 미션 실패(마감 초과) | -100 (잔액까지만) | 0으로 초기화 | 스케줄러 처리 |
